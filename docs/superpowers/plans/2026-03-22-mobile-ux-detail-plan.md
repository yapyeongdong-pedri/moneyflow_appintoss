# Money Flow 모바일 UX 상세 실행 플랜 (2026-03-22)

## 0) 이번 개편의 핵심 목표
- 기준 화면: 모바일 전용 (`390 x 844`) 프레임 우선.
- 진입 UX: 앱 설명 화면을 먼저 보여준 뒤 메인 화면 진입.
- 핵심 가치: 월급통장을 기준으로 Cash가 아래로 흐르는 구조를 한눈에 확인.
- 복잡도 제어: 메인에서 대부분 작업하되, 조작 버튼 수는 최소화.

## 1) 사용자 관점 화면 플로우
1. 첫 진입: 앱 설명 화면
- 앱의 목적을 10초 내 이해.
- "거래내역 추적"이 아니라 "현금 흐름 구조 점검" 앱이라는 점을 명확히 안내.
- CTA 1개 중심: `메인 화면으로 시작하기`.

2. 메인 화면: 월급통장 중심 Cash 흐름 캔버스
- 상단: 앱 타이틀 + 환경 배지 + 최소 액션(테마, 공유).
- 요약 카드: 현재 기준 계좌(월급통장)와 흐름 개수.
- 중심 영역: 세로 흐름 그래프.
- 하단 고정 액션: `흐름 정렬`, `노드 추가`, `상세 편집`.

3. 상세 편집 바텀시트
- 노드/엣지 선택 후 상세 편집.
- 이름 수정, 연결 라벨/메모, 활성 토글, 삭제.
- 보조 기능(실행 취소/다시 실행, PNG 저장)은 시트 내부로 이동.

## 2) 모바일 레이아웃 원칙
- 데스크톱에서도 모바일 프레임처럼 고정 노출.
- 실제 모바일에서는 풀스크린으로 자연 확장.
- Safe Area 하단 여백 확보 (`env(safe-area-inset-bottom)`).
- 과밀 방지: 상시 노출 액션은 3개 이하 유지.

## 3) Cash Flow 시각 구조 원칙
- 기본 정렬은 아래 순서를 따름:
  - 수입원 (좌상단)
  - 월급통장/자산계좌 (상단 중심)
  - 결제수단 (중단)
  - 지출/저축/투자/상환 (하단)
- 사용자가 레이아웃을 망가뜨려도 `흐름 정렬` 한 번으로 복구 가능.
- 라벨은 짧게, 의미는 명확히.

## 4) 메인 화면 복잡도 제어 체크리스트
- 상단 액션: 2~3개 유지.
- 플로팅 액션: 3개 유지.
- 보조 기능은 시트 내부로 이동.
- 메시지는 토스트 한 줄로 짧게.

## 5) 구현 단계 (실행 순서)
1. 모바일 프레임 고정 스타일 적용.
2. Intro -> Main 전환 UX 추가.
3. 월급통장 중심 기본 그래프(스타터) 배치.
4. 세로 흐름 자동 정렬 함수 적용.
5. 메인 액션 축소 및 상세 시트로 기능 재배치.
6. 저장/공유/PNG 동작 회귀 확인.

## 6) 검증 포인트
- 모바일 폭에서 잘리거나 겹치는 컴포넌트가 없는가.
- 첫 진입 시 설명 화면이 바로 뜨는가.
- 메인 진입 후 월급통장 중심 흐름이 하향식으로 보이는가.
- 메인 화면만으로 조회/추가/편집의 대부분이 가능한가.
- 버튼 수가 많아져 인지 부하가 커지지 않는가.

## 7) 다음 스프린트 권장 개선
1. 온보딩 중 "내 월급통장 이름" 1개만 먼저 받는 초간단 초기화.
2. 흐름 라인 두께를 금액 힌트(예: 높음/중간/낮음)로 표현.
3. 월 단위 리포트 카드(이번 달 고정비 경로 개수, 저축 경로 개수) 추가.
